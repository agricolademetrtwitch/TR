# 🤖 Steam Autonomous Multi-Account Control Panel v2026.GLOBAL

Полностью автономный веб-сайт и SaaS-система на **Node.js, Express, JavaScript и SQLite**, разработанная для круглосуточного (24/7) мониторинга, автоматического фарма коллекционных карточек, буста игровых часов и интеллектуальной защиты инвентаря от угроз типа **API-Scam (подмена трейдов)**.

Система спроектирована по модульному принципу и готова к развертыванию на облачных хостингах (Render, Railway, Heroku) или локальных серверах.

---

## 🚀 Ключевой Функционал

*   **Мультиаккаунтинговая Архитектура (Accounts Branch):** Добавление неограниченного количества Steam-ботов через веб-интерфейс. Каждый бот работает в изолированном потоке.
*   **Автономный Фарм 24/7:** Робот эмулирует запуск игр (CS2, TF2 и др.) для круглосуточного получения дропа и накрутки часов, не требуя включенного ПК или браузера у владельца.
*   **Интеллектуальный Анти-API Скам:** Система перехватывает входящие обмены. При обнаружении попытки скрытого/одностороннего вывода предметов (скам) трейд моментально отклоняется.
*   **ИИ-Движок Мутаций (Self-Evolution):** Интегрированный симулятор ИИ-ассистента, способный на лету пересчитывать налоговые шлюзы Габена (13.04%) и адаптировать лимиты Buy-ордеров при изменении рыночных условий.
*   **Постоянная База Данных SQLite:** Все логи, баланс, статусы ботов и конфигурации мутаций надежно сохраняются в файл БД и не стираются при перезагрузке сервера.

## 🛠️ Локальная установка и запуск

Если вы хотите запустить проект локально на своем компьютере:

1. Установите **Node.js** (версии 18.0.0 или выше) с официального сайта.
2. Скачайте файлы репозитория в пустую папку.
3. Откройте терминал/консоль в папке проекта и установите все зависимости:
   ```bash
   npm install
   ```
4. Запустите сервер автоматизации:
   ```bash
   npm start
   ```
5. Откройте браузер и перейдите по адресу: [http://localhost:3000](http://localhost:3000)

## 🌐 Инструкция по развертыванию в Облако (на примере Render.com)

Этот проект полностью оптимизирован под сетевые стандарты и облачные среды (динамические порты `process.env.PORT` и перенос БД в `/tmp` для обхода read-only ограничений).

1. Загрузите файлы `server.js`, `package.json` и папку `public` в ваш **GitHub** репозиторий.
2. Зарегистрируйтесь на **Render.com** через ваш GitHub-аккаунт.
3. Нажмите **New +** -> **Web Service** и выберите ваш репозиторий.
4. Выставите следующие параметры конфигурации:
   * **Name:** `любое-имя-вашего-сайта`
   * **Region:** `Frankfurt (EU)` (рекомендуется для стабильного пинга к серверам Steam)
   * **Language:** `Node`
   * **Build Command:** `npm install`
   * **Start Command:** `node server.js`
   * **Instance Type:** `Free`
5. Нажмите **Deploy Web Service**. По окончании сборки хостинг выдаст вам защищенную публичную `https://` ссылку на ваш сайт.

## 🔐 Безопасность Данных

*   **Изоляция Среды:** Ваши учетные данные передаются напрямую на бэкенд вашего собственного сервера. Проект не использует сторонние базы данных или серые API.
*   **SDA Совместимость:** Поле `Shared Secret` позволяет использовать хэш-ключи из Steam Desktop Authenticator для полностью автоматической генерации кодов Steam Guard при перезапуске сервера (без ручного ввода с телефона).
*   **Защита от SQL-инъекций:** Запросы к ветке базы данных SQLite обрабатываются с использованием безопасных подготовленных выражений (Prepared Statements).
